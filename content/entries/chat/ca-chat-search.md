### ❓ Что это

Поиск по прошлым чатам — Claude может находить и подтягивать релевантный контекст из прежних
разговоров пользователя, не требуя от него пересказывать заново, что обсуждалось раньше. Это
отдельная функция от долгосрочной Memory (которая хранит извлечённые факты) — chat search работает
именно с историей самих разговоров как с источником поиска, находя конкретные прошлые обсуждения по
смыслу запроса.

Разница между двумя механизмами принципиальна. Memory — это компактный слепок устойчивых фактов
(«пользователь предпочитает TypeScript»), который Claude держит наготове всегда, независимо от того,
о чём именно был разговор, в котором факт появился. Chat search, наоборот, не хранит выжимку — он
индексирует сами разговоры целиком и находит конкретный чат по смысловому совпадению с текущим
запросом, доставая из него оригинальный контекст: цитаты, код, решения, принятые именно в том
обсуждении. Это ближе к полнотекстовому/семантическому поиску по личному архиву переписки, чем к
памяти в привычном смысле.

### 🎯 Зачем тебе

Продолжение работы над темой, начатой недели назад: «мы это уже обсуждали, найди тот разговор про
архитектуру платёжного модуля и продолжи оттуда» — вместо ручного пролистывания истории чатов или
полного пересказа контекста заново, Claude сам находит релевантный прошлый разговор и опирается на
него.

Практический сценарий, где это особенно ценно — длинные проекты, растянутые на недели и месяцы, где
единого чата для всей темы не существует: решения принимались в разных разговорах, каждый по своему
подвопросу. Без поиска по истории пользователь либо держит в голове, в каком именно чате что
обсуждалось (и вручную его находит через интерфейс), либо заново объясняет контекст в каждом новом
разговоре. Chat search убирает это трение — достаточно сослаться на тему по смыслу, а не помнить
точную дату или название чата.

### 💻 Минимальный пример

```
Помнишь, мы обсуждали миграцию на JWT пару недель назад? Продолжи с того места
```

Claude ищет по истории чатов пользователя, находит релевантный разговор, подтягивает его содержание
как контекст для текущего ответа — без явной ссылки на конкретную дату или название чата от
пользователя.

### ⚠️ Грабли

- **Поиск по смыслу, не точному совпадению слов** — иногда полезно уточнить формулировку, если
  первая попытка не находит нужный разговор, как с любым семантическим поиском. Более специфичный
  запрос с реальными терминами из искомого разговора (названием модуля, именем участника, конкретной
  ошибкой) обычно находит точнее, чем расплывчатое «тот разговор про баг».
- **Не заменяет структурированную Memory** — chat search находит разговоры целиком, Memory хранит
  извлечённые устойчивые факты; для «Claude должен помнить это всегда» лучше подходит явное сохранение
  в Memory, а не расчёт на то, что поиск каждый раз найдёт нужный старый чат.
- **Зависит от настроек приватности/сохранения истории** — если история чатов не сохраняется по
  настройкам аккаунта, искать по ней нечего.
- **Очень старые или очень короткие разговоры хуже ранжируются** — как и в любом поиске по
  релевантности, объёмный содержательный разговор с чёткой темой находится надёжнее, чем короткий
  обмен репликами без явного тематического ядра.
