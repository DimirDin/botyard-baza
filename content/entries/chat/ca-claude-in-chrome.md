### ❓ Что это

Расширение для браузера, дающее Claude агентный доступ к открытой вкладке: видит содержимое страницы,
может кликать по элементам, заполнять формы, переходить по ссылкам — выполняя многошаговые задачи
прямо в реальном браузере пользователя, а не в изолированной песочнице. В отличие от Computer use
(который работает по скриншотам и координатам пикселей), расширение для Chrome DOM-aware — видит
структуру страницы напрямую, что быстрее и точнее для веб-специфичных задач.

Ключевое отличие от обычного агента с браузерным доступом через API/headless-инструменты — это
именно *твой* браузер с твоими сессиями. Claude действует в той же вкладке, где ты уже залогинен в
нужный сервис — не нужно передавать модели пароли или заново авторизовываться через отдельный
изолированный контекст. Это удобно, но одновременно означает, что агент по факту действует с
правами и доступом текущей сессии пользователя один в один, а не в урезанной песочнице.

### 🎯 Зачем тебе

Задачи, требующие реального взаимодействия с веб-сервисами от имени пользователя — заполнение форм
на сайтах без API, навигация по многошаговым веб-интерфейсам, сбор информации с нескольких страниц
подряд. Полезно там, где у сервиса нет удобного API, но есть обычный веб-интерфейс, которым
пользуется человек.

Типичный класс задач — рутина, которую человек и так делает руками через браузер, просто медленно:
сверка цен по товару на нескольких сайтах подряд, заполнение однотипной формы с данными из другого
источника, пролистывание многостраничного результата поиска с ручной фильтрацией. Там, где написание
отдельного скрипта под конкретный сайт не окупается разовой задачей, а API у сервиса просто нет,
агент в браузере закрывает разрыв между «это можно сделать руками» и «это стоит автоматизировать».

### 💻 Минимальный пример

После установки расширения и разрешения доступа к конкретной вкладке — запрос вида «найди на этом
сайте все товары дешевле 1000 рублей и сохрани список» заставляет Claude реально прокликать
страницу, прочитать содержимое DOM, собрать данные и вернуть структурированный результат.

### ⚠️ Грабли

- **Автономность на реальном браузере — реальный риск**, если задача касается финансовых операций
  или необратимых действий (отправка формы, покупка) — по политике безопасности такие действия либо
  требуют явного подтверждения, либо вообще не должны выполняться автономно без прямого участия
  человека на каждом шаге.
- **Не для непроверенных/подозрительных сайтов** — расширение с доступом к содержимому вкладки
  теоретически расширяет поверхность атаки через prompt injection со страницы (текст на странице,
  притворяющийся инструкцией для агента) — расширение спроектировано с защитой от этого, но общая
  осторожность с непроверенным контентом остаётся уместной.
- **Скорость ниже прямого API-вызова** — DOM-based автоматизация всё равно медленнее, чем
  специализированная интеграция через официальный API сервиса, если такой существует.
- **Права доступа — на уровне текущей вкладки/сессии, а не изолированный аккаунт** — агент действует
  с теми же куками и правами, что и залогиненный в браузере пользователь; выдавать расширению доступ
  стоит осознанно, а не по умолчанию на всех вкладках сразу.
