### ❓ Что это
Agentic Loop — внутренний итеративный цикл выполнения задач в Claude Code. Вместо «один промпт — один ответ» модель работает по схеме ReAct (Reasoning + Acting): разбивает задачу на подзадачи, вызывает инструменты, анализирует результат и корректирует действия, пока цель не будет достигнута.

### 🎯 Зачем тебе
Понимание цикла убирает иллюзию «магии» и позволяет прогнозировать поведение агента. Видно, на каком шаге он зациклился, когда стоит перехватить управление вручную — и перестать сжигать токены на пустые итерации.

### 💻 Минимальный пример
```
[Пользователь] ──> "Исправь баг в роуте авторизации"
        │
        ▼
1. THINK: нужно прочитать logs и router.py
2. ACT:   view_file
3. OBSERVE: содержимое файла
4. THINK: опечатка в JWT-валидации
5. ACT:   edit_file
6. OBSERVE: файл изменён
7. THINK: нужно проверить, что всё работает
8. ACT:   run_command pytest
9. OBSERVE: тесты прошли (exit 0)
        │
        ▼
[Claude Code] ──> "Баг исправлен, тесты зелёные"
```

### ⚠️ Грабли
Главная проблема — Tool Trap: если тесты падают из-за сломанного окружения (не кода), агент может уйти в цикл попыток чинить не то. Прерывай через Ctrl+C, если видишь одно и то же действие три раза подряд.
