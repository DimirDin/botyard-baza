### ❓ Что это

Внутренний цикл, на котором строится вся автономная работа Claude Code: модель получает текущее
состояние (запрос пользователя + история + результаты предыдущих действий), решает следующий шаг —
либо текстовый ответ, либо вызов инструмента (Read, Edit, Bash, Grep) — исполняет его, получает
результат обратно в контекст, и повторяет цикл, пока не сочтёт задачу решённой или не упрётся в
лимит/потребует подтверждения. Это тот же примитив «мысль → действие → наблюдение», что и в общем
паттерне ReAct, только с конкретными инструментами для работы с кодом.

### 🎯 Зачем тебе

Понимание этого цикла объясняет практическое поведение агента: почему он иногда делает «лишний» Read
перед правкой (проверяет актуальное состояние файла, а не доверяет памяти из более раннего шага
диалога), почему длинные задачи занимают много отдельных шагов вместо одного большого ответа, и
почему прерывание агента посреди цикла (Ctrl+C) оставляет код в промежуточном, а не откатанном
состоянии — цикл не транзакционен по своей природе.

### 💻 Минимальный пример

Схематично один проход цикла:

```
1. Модель: "нужно посмотреть текущую реализацию" → tool_use: Read(file)
2. Харнесс: исполняет Read, возвращает содержимое файла
3. Модель: анализирует содержимое → tool_use: Edit(file, old, new)
4. Харнесс: исполняет Edit, возвращает "успешно"
5. Модель: "нужно проверить, что тесты не сломались" → tool_use: Bash("npm test")
6. Харнесс: исполняет, возвращает вывод тестов
7. Модель: видит, что тесты прошли → финальный текстовый ответ, цикл завершён
```

### ⚠️ Грабли

- **Цикл не гарантирует конечность сам по себе** — без разумных ограничений (лимит шагов, permission
  prompts на опасные действия) агент теоретически может застрять в повторяющемся паттерне «попробовал
  → не сработало → попробовал похожее снова».
- **Каждый шаг цикла добавляет токены в контекст** — очень длинная сессия с десятками шагов упирается
  в те же проблемы context rot, что решают compaction и context editing.
- **Наблюдаемость цикла — не автоматическая прозрачность решений** — видно, *что* агент вызвал, но
  не всегда очевидно *почему* именно это, а не альтернативный путь — thinking-блоки (если включены)
  дают больше видимости в это рассуждение.
