### ❓ Что это

Официальный свод практик Anthropic по эффективной работе с Claude Code, сведённый к нескольким
принципам: держать `CLAUDE.md` с актуальным контекстом проекта (архитектура, команды, стиль кода —
не то, что и так видно из кода); придерживаться цикла explore → plan → code → commit вместо
«сразу пиши код»; формулировать узкие, конкретные задачи вместо расплывчатых «сделай лучше»;
активно курировать контекст — не бояться `/clear` между несвязанными задачами, чтобы не тащить
нерелевантную историю.

#### Explore-plan-code-commit

Порядок неслучаен: сначала агент должен реально прочитать релевантный код (explore), затем
предложить план изменений (plan, см. Plan mode), только потом писать код (code), и в конце
зафиксировать результат осмысленным коммитом (commit) — пропуск первых двух шагов на сложных
задачах статистически даёт больше неверных предположений о структуре проекта.

### 🎯 Зачем тебе

Прямое повышение качества результата без изменения самого инструмента — те же тулы, но
структурированный процесс работы с ними снижает долю «агент написал код, не понимая контекста».
Особенно ощутимо на больших незнакомых кодовых базах.

### 💻 Минимальный пример

Вместо `«почини баг с логином»` — `«в src/auth/session.ts баг: сессия истекает раньше TTL, указанного
в конфиге. Сначала найди причину, покажи план фикса, потом реализуй»` — явный путь через
explore-plan-code для конкретной, узко очерченной задачи.

### ⚠️ Грабли

- **Расплывчатые задачи дают расплывчатые результаты** — «сделай лучше» без критерия «лучше» почти
  гарантированно потребует повторной итерации.
- **Раздутый CLAUDE.md с общеизвестной информацией** («это React-проект», видно из package.json) —
  тратит контекст впустую, файл должен содержать то, что *нельзя* вывести из кода за секунды.
- **Игнорирование `/clear` между несвязанными задачами** — накопленная нерелевантная история от
  предыдущей задачи путает агента при работе над следующей.
