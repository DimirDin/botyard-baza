### ❓ Что это

Claude Code автоматически снимает состояние кода перед каждой существенной правкой в рамках сессии —
это не git commit (не создаёт видимую историю в репозитории), а внутренний снапшот, к которому можно
вернуться командой rewind, если серия правок агента пошла не туда. Работает независимо от того,
закоммичены ли текущие изменения в git.

### 🎯 Зачем тебе

Страховка для длинных автономных сессий: агент сделал 15 правок подряд, на 12-ой свернул не туда — без
checkpointing единственный способ откатиться это ручной git reset (если вообще есть чистая точка для
отката) или разбор диффа руками. С checkpointing — команда rewind к состоянию до конкретной правки.

### 💻 Минимальный пример

```
> /rewind
```

Показывает список последних чекпоинтов сессии с кратким описанием, к какой правке они относятся —
выбираешь нужную точку, код физически возвращается к тому состоянию, диалог продолжается дальше уже
с учётом отката.

### ⚠️ Грабли

- **Не заменяет git** — checkpointing живёт в рамках текущей сессии Claude Code, для долгосрочной
  истории и совместной работы всё равно нужны нормальные коммиты.
- **Откат кода не откатывает побочные эффекты вне файловой системы** — если правка агента успела,
  например, отправить запрос во внешний API или изменить БД, checkpointing вернёт только файлы.
- **Чекпоинты не переживают завершение сессии бессрочно** — не полагайся на них как на постоянный
  архив, это оперативная страховка на время работы, а не замена системы контроля версий.
- **Rewind диалога и rewind кода — не обязательно одно и то же действие** — уточняй, что именно
  откатываешь: только состояние файлов, только историю разговора, или оба сразу, если инструмент
  различает эти варианты, иначе легко откатить больше или меньше, чем на самом деле хотел.
