### ❓ Что это

`CLAUDE.md` — главный конфигурационный файл локального контекста проекта, который Claude Code читает
автоматически перед стартом любой задачи в репозитории. Это не документация для людей (README) и не
подробное техническое ТЗ — это компактный набор фактов, которые нельзя быстро вывести чтением самого
кода: команды сборки/тестов, архитектурные ограничения («не менять схему БД без миграции»), стиль
кода специфичный для проекта, важные факты, которые уже находили и фиксировали, чтобы не переоткрывать
заново.

### 🎯 Зачем тебе

Без `CLAUDE.md` каждая новая сессия агента начинает с нуля — заново гадает, как запустить тесты, какой
пакетный менеджер использовать, какие файлы никогда не трогать. С хорошим `CLAUDE.md` эта информация
подгружается автоматически при старте, экономя и токены на исследование, и время на повторные ошибки
по одним и тем же граблям.

Особенно ценен файл там, где знание не выводится из кода механически — правило вида «не менять схему
БД без миграции» не прочитать в самом коде схемы, это организационное соглашение, которое либо было
где-то устно передано, либо не соблюдается вовсе. `CLAUDE.md` — способ закрепить именно такие
негласные правила проекта в явном, машиночитаемом месте, доступном каждой новой сессии одинаково,
вместо того чтобы полагаться на то, что человек лично о них напомнит каждый раз.

### 💻 Минимальный пример

```markdown
# CLAUDE.md

## Build & Test
\`\`\`bash
cd backend && pip install -r requirements.txt
pytest
\`\`\`

## Важные факты
- Схема БД называется `baza`, никогда не пиши в неё контент напрямую мимо `scripts/sync_content.py`
- Гейт-логику (`app/gate.py`) не менять без сверки с внешним контекст-документом
```

### ⚠️ Грабли

- **Раздутый файл с очевидностями** («используем Python» — видно из `requirements.txt`) тратит
  контекст каждой сессии впустую, держи только то, что реально неочевидно.
- **Устаревший CLAUDE.md хуже отсутствующего** — если файл описывает архитектуру, которая уже
  изменилась, агент будет действовать по неверным предпосылкам увереннее, чем если бы просто
  исследовал код сам.
- **Не место для секретов** — `CLAUDE.md` обычно коммитится в репозиторий, никаких токенов и паролей
  внутри, только `.env.example` с пустыми значениями рядом.
