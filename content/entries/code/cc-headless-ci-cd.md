### ❓ Что это

Headless-режим запускает Claude Code без интерактивного терминального UI — через флаг командной
строки (`claude -p "промпт"` или подобный неинтерактивный вызов), что делает его пригодным для
встраивания в CI/CD пайплайны: автоматический код-ревью на каждый PR, генерация тестов на push,
рефакторинг по расписанию — весь процесс идёт скриптом без ожидания ручного ввода на permission
prompts.

### 🎯 Зачем тебе

Любая автоматизация, где Claude Code должен отработать как шаг пайплайна, а не как интерактивный
собеседник — GitHub Actions job, Jenkins step, cron-задача на CI-сервере. Headless-режим — фундамент,
на котором построена и интеграция GitHub Actions (`@claude` в PR), просто более общий и гибкий
интерфейс для произвольных скриптовых сценариев.

Практически это открывает Claude Code для любой CI-системы, а не только GitHub — Jenkins, GitLab CI,
самописный раннер, всё, что умеет выполнить shell-команду и распарсить вывод. Готовая интеграция с
GitHub Actions (см. отдельную статью) удобна там, где репозиторий уже на GitHub, но `-p` флаг —
универсальный примитив снизу, на котором можно построить эквивалентную автоматизацию где угодно.

### 💻 Минимальный пример

```bash
claude -p "проверь этот diff на security-проблемы, выведи JSON с находками" \
  --output-format json > review.json
```

В CI-скрипте результат парсится как обычный JSON-вывод любого другого шага пайплайна — CI не отличает
семантически «агент подумал» от «скрипт посчитал», просто получает структурированный результат.

### ⚠️ Грабли

- **Нет интерактивных permission prompts в headless-режиме** — для потенциально опасных действий
  нужна явная предварительная настройка permissions/sandboxing, иначе либо агент заблокирован на
  запрещённых по умолчанию операциях, либо (при неверной настройке) выполняет их без подтверждения.
- **Таймауты CI-раннеров конечны** — долгая агентная сессия может упереться в лимит времени job'а,
  закладывай реалистичный бюджет времени под сложность задачи.
- **Стоимость токенов в CI на каждый PR/push складывается** — headless-автоматизация на большом
  потоке коммитов может дать заметный счёт, стоит ограничивать триггеры разумными условиями
  (явный `@claude`, а не каждый push подряд).
