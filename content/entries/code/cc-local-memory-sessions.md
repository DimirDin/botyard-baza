### ❓ Что это

Claude Code сохраняет историю и контекст сессий локально между перезапусками терминала — можно
вернуться к прерванному разговору командой возобновления сессии, не пересказывая заново, что уже
обсуждалось и что было сделано. Это отдельный механизм от auto-memory (персистентные факты о
пользователе/проекте между *разными* сессиями) — здесь речь именно про продолжение *одной и той же*
сессии после закрытия терминала.

### 🎯 Зачем тебе

Реалистичный рабочий день с прерываниями — закрыл ноутбук посреди задачи, вернулся через час, хочешь
продолжить с того же места без потери контекста уже сделанных правок и обсуждений. Без сохранения
сессий каждый перезапуск терминала означал бы полную потерю истории разговора.

### 💻 Минимальный пример

```bash
claude --resume        # выбор из списка последних сессий
claude --continue       # продолжить самую последнюю сессию без выбора
```

### ⚠️ Грабли

- **Локальное хранилище сессий растёт со временем** — при большом числе долгих сессий стоит
  периодически чистить старые, если место на диске становится заметным ограничением.
- **Возобновлённая сессия — это восстановленный контекст, не восстановленное состояние файловой
  системы** — если между закрытием терминала и возобновлением файлы менялись извне (другим
  инструментом, коллегой), агент может опираться на устаревшее представление, пока явно не
  перечитает актуальное состояние.
- **Не путать с checkpointing** — локальная память сессий про продолжение диалога, checkpointing про
  откат уже сделанных правок кода, это разные механизмы с разным назначением.
