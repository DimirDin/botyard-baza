### ❓ Что это
Интеграция cc с локальным демоном Docker через MCP даёт агенту типизированные инструменты для сборки образов, просмотра статуса контейнеров, чтения логов и перезапуска — прямо из текущей сессии.

### 🎯 Зачем тебе
Не нужно переключаться между терминалом и соседними вкладками с `docker compose logs -f`. Можно попросить агента самому найти причину падения контейнера и поправить конфиг.

### 💻 Минимальный пример
Официальный способ — через CLI-команду `claude mcp add`, а не ручное редактирование конфига:
```bash
claude mcp add docker -- npx -y <docker-mcp-пакет>
claude mcp list   # проверить статус подключения
```
Для командной работы — зафиксируй в `.mcp.json` в корне проекта (коммитится в git, применяется всей команде):
```json
{
  "mcpServers": {
    "docker": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "<docker-mcp-пакет>"]
    }
  }
}
```
Точное имя npm-пакета проверь в актуальном каталоге MCP-серверов перед использованием — экосистема быстро меняется, конкретные пакеты появляются и устаревают.

### ⚠️ Грабли
Проброс доступа к Docker-сокету даёт агенту фактически root-права на хосте. Деструктивная команда очистки может снести все локальные базы и кэши образов. Не оставляй агента в headless-режиме с доступом к Docker без присмотра.
