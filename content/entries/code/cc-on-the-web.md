### ❓ Что это

Claude Code on the web запускает задачи на облачной инфраструктуре Anthropic, а не на локальной
машине разработчика — ты ставишь задачу через веб-интерфейс или мобильное приложение, агент работает
в изолированном облачном окружении с доступом к твоему репозиторию, а ты можешь мониторить прогресс
удалённо, включая с телефона, и при необходимости «телепортироваться» в полноценный терминальный
сеанс прямо посреди облачной сессии для ручного вмешательства.

### 🎯 Зачем тебе

Задачи, которые не требуют твоего локального окружения (нет специфичных для машины зависимостей,
секретов, которые нельзя выдать облаку) и которые ты хочешь запустить и не ждать у экрана —
долгий рефакторинг, прогон большого набора тестов с последующим фиксом, обработка issue из багтрекера
в фоне, пока ты занят другим. Мониторинг с телефона превращает Claude Code из «инструмента, требующего
открытого терминала» в асинхронный сервис.

### 💻 Минимальный пример

Через веб-интерфейс: выбираешь репозиторий, формулируешь задачу («почини все упавшие тесты в CI»),
запускаешь — сессия идёт в облаке, статус доступен в UI, при готовности результата (обычно в виде
PR или диффа) приходит уведомление; в любой момент можно «телепортироваться» — открыть эту же сессию
в локальном терминале, продолжив работу интерактивно.

### ⚠️ Грабли

- **Секреты и локальные интеграции недоступны облаку по умолчанию** — если задача требует доступа к
  внутренней сети компании или локальным credentials, облачная сессия либо не справится, либо нужна
  явная настройка secrets management.
- **Не заменяет ревью** — облачный результат (даже оформленный как PR) требует того же ревью, что и
  локальная работа агента, автономность выполнения не равна автоматическому доверию к результату.
- **Задержка/доступность зависят от облачной капасити Anthropic**, а не от твоего железа — для очень
  срочных задач локальный запуск может быть предсказуемее по времени старта.
