### ❓ Что это

Output styles меняют роль, тон и формат ответов Claude Code — переключают агента из режима «пишет
код» в другие режимы работы: writing-ассистент для текстов, аналитик, объясняющий вместо
исполняющий, стиль «всегда с диаграммой» для архитектурных решений. Это не косметическая настройка
цвета вывода в терминале, а изменение самого поведения модели через модифицированный системный
промпт стиля.

### 🎯 Зачем тебе

Claude Code — не только про код: ревью юридического документа, разбор бизнес-плана, объяснение
концепции для нетехнического человека. Дефолтный стиль оптимизирован под кодинг (краткость,
минимум объяснений, сразу к делу) — для других задач это не всегда лучший тон, output styles дают
переключатель без необходимости каждый раз формулировать инструкцию по стилю вручную в промпте.

### 💻 Минимальный пример

```
/output-style explanatory
```

Переключает на режим, где агент подробнее объясняет свои решения по ходу работы — полезно при
обучении или когда нужно понять «почему именно так», а не только получить готовый результат.

### ⚠️ Грабли

- **Стиль не меняет доступные инструменты** — это про тон и формат ответа, не про то, какие тулы
  агенту разрешены (это регулируется permissions отдельно).
- **Не все стили одинаково хорошо сочетаются с автономной долгой работой** — многословный
  объяснительный стиль в длинной агентной сессии раздувает контекст быстрее, чем лаконичный.
- **Стиль применяется на уровне сессии**, не запроса — если нужно разное поведение в рамках одного
  разговора, переключение туда-обратно возможно, но не мгновенно бесшовно.
