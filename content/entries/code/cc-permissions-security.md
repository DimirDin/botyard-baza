### ❓ Что это
Слой Permissions ограничивает права агента на деструктивные действия. По умолчанию cc может читать и менять файлы в текущей директории, но для bash-команд, установки пакетов или сетевых запросов требуется явное подтверждение.

### 🎯 Зачем тебе
Без контроля доступов ошибка в сгенерированном скрипте может привести к выполнению опасной команды или перезаписи системных конфигов. Permissions создают безопасную песочницу для .env, SSH-ключей и остальной системы.

### 💻 Минимальный пример
```
[cc] Запрашивает разрешение на выполнение:
$ poetry run alembic upgrade head

[Y] Разрешить один раз
[A] Разрешать всегда в этой сессии
[N] Отклонить
[Q] Выйти
```

### ⚠️ Грабли
Никогда не отключай ручные подтверждения в проектах с непроверенными сторонними зависимостями — агент может прочитать и выполнить вредоносный скрипт из node_modules или venv без твоего ведома.
