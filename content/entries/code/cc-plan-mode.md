### ❓ Что это

Plan mode — режим Claude Code, в котором агент сначала исследует кодовую базу (читает файлы, ищет
паттерны, смотрит зависимости) и предлагает текстовый план действий, **ничего не меняя на диске**,
пока ты явно не одобришь этот план. Только после одобрения агент переходит к фактическим правкам —
Edit/Write/Bash с реальными side-эффектами.

Переключается через Shift+Tab или явный запрос «сначала распланируй». В режиме плана инструменты
записи заблокированы на уровне харнеса — это не «агент обещает не трогать файлы», а системное
ограничение доступных тулов на время режима.

### 🎯 Зачем тебе

Любая задача, где неверная интерпретация запроса дорого стоит переделкой — большой рефакторинг,
правка в незнакомой части кодовой базы, миграция схемы БД. План — это дешёвая точка сверки: прочитать
5-10 строк текста и поправить направление куда дешевле, чем откатывать полсотни изменённых файлов.

### 💻 Минимальный пример

```
> сначала распланируй, потом делай: перенеси авторизацию с сессий на JWT
```

Агент читает текущую реализацию auth, находит все точки использования сессий, и возвращает план вида:
«1. Добавить JWT-миддлварь. 2. Заменить `session.user` на `req.user` в 12 местах. 3. Обновить тесты.
4. Убрать зависимость от express-session.» — ты можешь попросить поправить план до одобрения.

### ⚠️ Грабли

- **План не заменяет ревью финального диффа** — план описывает намерение, а не гарантирует точную
  реализацию, всё равно смотри итоговые изменения.
- **Слишком общий запрос даёт слишком общий план** — «сделай лучше» вернёт расплывчатый план;
  конкретная формулировка задачи даёт конкретный, проверяемый план.
- **Не путай с checkpointing** — план mode предотвращает нежелательные правки заранее, checkpointing
  откатывает уже сделанные правки постфактум, это дополняющие, не взаимозаменяемые механизмы.
