### ❓ Что это

Плагин Claude Code упаковывает вместе несколько типов расширений — слэш-команды, суб-агентов,
хуки, MCP-серверы и скиллы — в единый устанавливаемый пакет с версионированием. Маркетплейс —
источник, откуда такие плагины устанавливаются (по аналогии с npm-реестром, только для расширений
Claude Code), может быть публичным сообщенческим или приватным корпоративным.

### 🎯 Зачем тебе

Готовые пакеты вместо ручной сборки из отдельных кусков: вместо того чтобы самому писать хук для
линтинга, MCP для конкретного стека и набор слэш-команд под него по отдельности, устанавливаешь один
плагин, который уже включает всё это согласованным набором. Для команд — способ распространить
единый рабочий процесс на всех разработчиков одной установкой вместо синхронизации конфигов вручную.

### 💻 Минимальный пример

```
/plugin marketplace add my-org/internal-plugins
/plugin install code-review-suite
```

После установки становятся доступны команды, суб-агенты и хуки, которые определил плагин — без
ручного копирования файлов конфигурации в каждый проект по отдельности.

### ⚠️ Грабли

- **Плагин из непроверенного маркетплейса — риск supply chain** — хуки и MCP-серверы плагина
  исполняют код на твоей машине, устанавливай только из доверенных источников, как с любым пакетным
  менеджером.
- **Версии плагинов могут конфликтовать** — если два установленных плагина определяют команду с
  одинаковым именем, поведение переопределения зависит от порядка установки, лучше избегать
  пересечений явно.
- **Приватный корпоративный маркетплейс требует инфраструктуры для хостинга** — это не встроенная
  бесплатная фича, а отдельная настройка на стороне организации.
