### ❓ Что это

Remote MCP — подключение MCP-серверов по HTTP/SSE вместо запуска локального процесса на машине
разработчика. Сервер живёт где-то в облаке (у провайдера сервиса или на собственной инфраструктуре
компании), Claude Code подключается по URL, аутентификация обычно через OAuth — без необходимости
устанавливать и поддерживать локальный биндинг под каждую интеграцию.

### 🎯 Зачем тебе

Интеграции с SaaS-сервисами, у которых есть готовый remote MCP-сервер (Slack, Linear, корпоративные
внутренние системы) — не нужно поднимать и обновлять локальный процесс, авторизация проходит один раз
через OAuth-флоу, дальше сервер доступен во всех твоих сессиях Claude Code без переустановки. Особенно
полезно для команд — один и тот же remote MCP настраивается централизованно для всех разработчиков.

### 💻 Минимальный пример

```json
{
  "mcpServers": {
    "linear": { "type": "url", "url": "https://mcp.linear.app/sse" }
  }
}
```

При первом обращении Claude Code откроет OAuth-флоу авторизации в браузере, дальше токен кэшируется
локально и переиспользуется между сессиями до истечения.

### ⚠️ Грабли

- **Требует сетевого доступа наружу** — в изолированных корпоративных сетях без выхода в интернет
  remote MCP недоступен, там нужен локальный сервер или проксирование.
- **Задержка сети добавляется к каждому вызову тула** — remote MCP медленнее локального процесса на
  каждый round-trip, для высокочастотных вызовов это заметно.
- **OAuth-токены нужно ротировать и следить за их истечением** — просроченный токен ломает
  интеграцию молча до следующей попытки использования, если нет явного уведомления об ошибке.
- **Доверие к серверу переходит на его оператора** — в отличие от локального MCP, код которого ты
  контролируешь, remote-сервер исполняется на чужой инфраструктуре; подключай только те сервисы,
  которым готов доверить то, что видит агент в рамках сессии.
