### ❓ Что это

Sandboxing — ОС-уровневая изоляция файловой системы и сети для Bash-тула Claude Code: агент
исполняет команды внутри ограниченного окружения (например, без доступа к файлам вне рабочей
директории, без произвольного сетевого доступа за пределы разрешённого списка), вместо того чтобы
каждая потенциально опасная команда требовала ручного подтверждения человеком в терминале.

### 🎯 Зачем тебе

Без песочницы автономная работа Bash tool — это постоянные permission prompts на каждую команду,
которая формально могла бы быть опасной (запись файла, сетевой запрос) — быстро становится узким
местом при долгой агентной сессии. С правильно настроенным sandboxing агент работает автономнее
внутри безопасных границ, а подтверждение спрашивается только для действий, реально выходящих за
эти границы.

### 💻 Минимальный пример

Настройка (концептуально) — allow-list разрешённых путей и доменов в конфиге проекта, дальше Bash
tool исполняется в среде, где `rm -rf /` физически невозможен (нет доступа за пределы sandbox), а
`npm install` работает, потому что сеть к registry явно разрешена.

### ⚠️ Грабли

- **Sandboxing — не абсолютная защита**, а снижение поверхности атаки — правильно настроенный
  allow-list всё ещё может быть слишком широким для реально чувствительных проектов.
- **Слишком узкий sandbox ломает легитимную работу** — если агенту нужен доступ к системному
  пакетному менеджеру или внешнему сервису, а его нет в allow-list, задача просто не выполнится, и
  дебаг «почему не работает» усложняется на слой изоляции.
- **Не заменяет ревью кода** — изолированное исполнение не гарантирует, что сгенерированный код
  корректен, только что он не может навредить системе за пределами песочницы во время выполнения.
