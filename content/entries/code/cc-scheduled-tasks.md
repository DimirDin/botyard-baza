### ❓ Что это

Связка инструментов для повторяющейся и отложенной работы: `/loop` повторяет промпт или слэш-команду
с заданным интервалом (или с самопланируемым темпом в динамическом режиме), cron-тулы позволяют
поставить задачу на конкретное время или регулярное расписание — «проверяй статус деплоя каждые 5
минут», «раз в день по утрам собери отчёт по open issues».

### 🎯 Зачем тебе

Задачи, требующие периодической проверки состояния без постоянного присутствия человека за терминалом:
поллинг долгого CI-пайплайна, регулярный мониторинг метрики с алертом при отклонении, автоматический
дневной дайджест активности в репозитории. Без scheduled tasks это требует либо ручного возврата в
терминал по таймеру, либо отдельной внешней инфраструктуры cron просто ради вызова Claude Code.

### 💻 Минимальный пример

```
/loop 5m проверь статус деплоя и сообщи, когда он завершится
```

Для одноразового отложенного запуска — постановка задачи с конкретным временем срабатывания вместо
интервала, что удобно для «напомни мне об этом завтра в 9 утра» сценариев.

### ⚠️ Грабли

- **Не подходит для очень частого опроса** — polling короче минуты дороже по накладным расходам, чем
  просто дождаться уведомления о завершении отслеживаемого процесса, если такое доступно.
- **Долгий /loop без явного условия остановки** может работать намного дольше, чем предполагалось —
  всегда задавай явный критерий завершения или максимальное число итераций.
- **Расписание не переживает завершение процесса/сессии автоматически** без явно настроенного
  персистентного механизма (см. cron-инструменты отдельно от простого `/loop`), это не системный
  демон по умолчанию.
