### ❓ Что это

Авторинг Agent Skills — создание собственных `SKILL.md`-файлов, которые упаковывают повторяющийся
рабочий процесс в переиспользуемую команду. Структура строится на принципе прогрессивного раскрытия:
короткое имя и `description`-триггер (когда Claude должен сам решить подключить этот скилл),
основное тело инструкции (что именно делать), и опциональные вложенные ресурсы — шаблоны, скрипты,
справочные файлы, которые подгружаются только когда реально нужны, а не сразу целиком в контекст.

#### Прогрессивное раскрытие

Ключевая идея: `description` — единственное, что постоянно занимает место в контексте (используется
для решения, релевантен ли скилл текущему запросу), полное тело `SKILL.md` подгружается только при
активации, а вложенные файлы — только если сам скилл явно к ним обращается по ходу выполнения. Это
позволяет иметь десятки скиллов, не раздувая базовый контекст каждой сессии.

### 🎯 Зачем тебе

Любой процесс, который ты формулируешь одинаково в промпте раз за разом — «напиши тесты в моём
стиле», «сгенерируй отчёт по этому шаблону», «проверь код по нашему чек-листу безопасности».
Оформленный как скилл, он становится триггерящимся автоматически по описанию или явно вызываемым
через `/имя-скилла`, вместо того чтобы копипастить длинную инструкцию в каждый новый разговор.

### 💻 Минимальный пример

```markdown
---
name: write-tests-in-my-style
description: Используй при написании unit-тестов для этого проекта — задаёт конвенции именования и структуру AAA
---

Пиши тесты по паттерну Arrange-Act-Assert, именование `test_<функция>_<условие>_<ожидание>`,
используй фикстуры из `tests/fixtures.py`, не мокай внутренние функции модуля — только внешние
зависимости (БД, сеть).
```

### ⚠️ Грабли

- **Расплывчатый `description` не триггерится автоматически** — если описание не содержит явных
  ключевых слов ситуации применения, модель не подключит скилл сам, только по явному вызову.
- **Слишком длинное тело SKILL.md без вложенных файлов** — теряется весь смысл прогрессивного
  раскрытия, если всё содержимое грузится сразу целиком при каждой активации.
- **Конфликтующие по смыслу описания у разных скиллов** — модель может выбрать не тот скилл, если
  два триггерятся на похожие ситуации без явного разграничения.
