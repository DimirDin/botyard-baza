### ❓ Что это

Разница между локальными скиллами и внешними плагинами/MCP-серверами как способами расширения
возможностей Claude Code. Skill — это `SKILL.md`-файл с инструкцией повторяемого рабочего процесса,
живущий локально в проекте или домашней директории пользователя, активируется по описанию или явному
вызову. Plugin — упакованный набор из нескольких типов расширений сразу (команды, суб-агенты, хуки,
MCP, скиллы), устанавливаемый из маркетплейса как единый версионированный пакет. MCP-сервер — вообще
отдельная категория, внешний процесс/сервис, дающий агенту новые *инструменты* (не инструкции, а
реальные API-вызовы к внешним системам).

### 🎯 Зачем тебе

Выбор правильного механизма под задачу: если нужно закрепить *способ выполнения* задачи («всегда
пиши тесты вот так») — это skill. Если нужно распространить *готовый комплексный набор* расширений
на всю команду одной установкой — это plugin. Если нужен *доступ к внешней системе* (БД, API,
файловое хранилище) — это MCP-сервер, отдельно от skill/plugin как категория.

### 💻 Минимальный пример

| Механизм | Что даёт | Пример |
|---|---|---|
| Skill | Инструкцию/процесс | «Пиши коммиты в conventional commits формате» |
| Plugin | Пакет из нескольких расширений | Комплект: hook линтера + subagent ревьюер + skill |
| MCP | Реальный инструмент к внешней системе | Чтение таблиц из Postgres |

### ⚠️ Грабли

- **Skill не даёт новых возможностей, только направляет использование существующих** — если задача
  требует доступа к системе, которого у агента физически нет, никакой skill это не решит, нужен MCP.
- **Plugin может незаметно принести MCP-сервер с сетевым доступом** — устанавливая плагин из
  маркетплейса, проверяй, что именно он включает, а не только заявленное название.
- **Пересекающиеся по смыслу skill из разных источников** конфликтуют по приоритету активации —
  избегай дублирования одной и той же инструкции в нескольких местах.
- **Три механизма легко смешать концептуально, но они решают ортогональные задачи** — вопрос «мне
  нужен skill, plugin или MCP» почти всегда на самом деле два независимых вопроса сразу: «нужен ли
  новый *доступ*» (MCP) и «нужно ли закрепить *способ работы*» (skill), а plugin — просто удобная
  упаковка для распространения любой комбинации первых двух командой.
