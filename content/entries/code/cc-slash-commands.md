### ❓ Что это

Встроенные команды Claude Code, начинающиеся с `/`, для быстрых операций без формулировки полного
промпта: `/compact` (сжать текущий контекст вручную, не дожидаясь автоматического порога), `/bug`
(сообщить о проблеме в самом Claude Code), `/review` (запустить ревью текущего diff'а), `/clear`
(начать разговор с чистого контекста в том же терминале), `/undo`/`/rewind` (откат последних правок,
см. Checkpointing).

### 🎯 Зачем тебе

Экономия токенов и времени на рутинных операциях, для которых не нужно формулировать текстовый
промпт — команда либо ничего не стоит по токенам (чисто клиентское действие), либо стоит
предсказуемо меньше произвольного текстового запроса на ту же операцию. Хороший быстрый справочник
для новых пользователей — не нужно запоминать, «как попросить агента сжать контекст словами», если
есть прямая команда.

Помимо экономии, встроенные команды предсказуемы ровно потому, что не проходят через интерпретацию
модели — `/clear` всегда делает одно и то же одинаково, а текстовая просьба «начни новый разговор»
теоретически может быть понята чуть иначе в зависимости от остального контекста сообщения. Для
операций, где точность важнее гибкости формулировки (сжатие контекста, откат, смена режима), прямая
команда — более надёжный инструмент, чем естественноязыковой запрос той же цели.

### 💻 Минимальный пример

```
/compact   — принудительное сжатие истории прямо сейчас
/clear     — новый разговор в том же терминале, старый контекст не тянется
/review    — ревью текущего git diff
```

### ⚠️ Грабли

- **`/clear` не откатывает уже сделанные правки файлов** — это только про контекст диалога, код
  остаётся в текущем состоянии, для отката кода нужен `/rewind` или git.
- **Список команд может отличаться между версиями Claude Code** — не все команды одинаково доступны
  во всех режимах (например, некоторые недоступны в headless-режиме), сверяйся с актуальной справкой
  своей версии, а не с запомненным списком.
- **`/compact` вручную посреди важного рассуждения** может преждевременно потерять нюансы, если
  вызван не в естественной паузе задачи — предпочтительно дожидаться завершения логического этапа.
