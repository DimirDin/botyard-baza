### ❓ Что это

Git worktrees позволяют иметь несколько рабочих директорий одного репозитория одновременно, каждая
на своей ветке, без переключения контекста через `git checkout` в единственной копии. Claude Code
использует это для параллельных сессий агента: одна сессия работает над фичей в одном worktree,
другая одновременно чинит баг в другом — без коллизий файловой системы и без риска, что правки одной
сессии затрут незакоммиченные правки другой.

### 🎯 Зачем тебе

Классическая проблема одной рабочей копии: пока Claude Code работает над долгой задачей А, ты не
можешь параллельно попросить его (или запустить вторую сессию) на задачу Б в том же репозитории без
конфликта незакоммиченных изменений. Worktrees решают это физически — разные директории на диске,
разные ветки, полностью изолированные друг от друга рабочие состояния.

Это особенно ощутимо при работе с длинными автономными сессиями — если фича делается 20-30 минут без
постоянного наблюдения, простой второй сессии всё это время означает потерянное параллельно время.
Worktrees превращают одну машину разработчика в подобие нескольких независимых рабочих мест: можно
буквально запустить несколько задач одновременно, каждую в своём терминале, и переключаться между
ними по мере готовности, вместо строго последовательного выполнения одной за другой.

### 💻 Минимальный пример

```bash
git worktree add ../repo-feature-x feature-x
cd ../repo-feature-x
claude   # отдельная сессия, отдельная ветка, отдельная директория
```

Каждая директория — полноценная рабочая копия репозитория со своим `.git` указателем на общий объект-
хранилище, изменения в одной не видны в файловой системе другой до explicit push/pull между ветками.

### ⚠️ Грабли

- **Каждый worktree — отдельная установка зависимостей** — `node_modules`/venv не расшариваются
  автоматически между worktree, это реальные накладные расходы на диск и время настройки.
- **Не забывай `git worktree remove`** после завершения работы — заброшенные worktree копятся и
  засоряют вывод `git worktree list`, путая будущие сессии.
- **Одновременная работа над одной и той же веткой в двух worktree одновременно невозможна** — git
  не даёт зачекаутить одну ветку в двух местах разом, worktrees про параллельность *разных* веток.
