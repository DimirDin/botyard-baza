### ❓ Что это

Hooks — обработчики событий жизненного цикла Claude Code: скрипт или команда, которые срабатывают
на конкретный момент (перед вызовом инструмента, после него, в начале сессии и т.д.) независимо от
того, что «решила» модель. Каждый hook — это связка Событие → Matcher (regex-фильтр, какие именно
вызовы ловим) → Действие (обычно shell-команда, но может быть промпт для лёгкой модели или
саб-агент).

### 🎯 Зачем тебе

Форматирование кода после каждой правки, блокировка опасных команд (`rm -rf`) до их выполнения,
запрет коммита, пока не прошли тесты — всё это работает гарантированно каждый раз, а не зависит от
того, вспомнит ли Claude запустить линтер. `PreToolUse` может одобрить или заблокировать действие
до выполнения, `PostToolUse` — проверить и поправить результат уже после.

### 💻 Минимальный пример

Автоформатирование после любой правки файла (`.claude/settings.json`):
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit|MultiEdit",
        "hooks": [
          { "type": "command", "command": "npx prettier --write \"$CLAUDE_TOOL_INPUT_FILE_PATH\"" }
        ]
      }
    ]
  }
}
```

### ⚠️ Грабли

`PostToolUse` может дать Claude обратную связь, но не может отменить уже выполненное действие —
для блокирующей защиты (запретить опасную команду) нужен именно `PreToolUse`. Общие для команды
правила стоит класть в `.claude/settings.json` проекта, а не в личный конфиг — иначе гарантии
работают только у тебя, а не у всей команды.
