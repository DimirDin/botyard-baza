### ❓ Что это

Без явных указаний Claude тяготеет к «дефолтному» виду интерфейса — стандартные отступы, типовая
палитра, привычные шрифты. Это можно исправить двумя путями: (1) явно задавать в промпте
дизайн-токены — конкретные HEX-цвета, шрифтовую пару, шкалу отступов, а не «сделай красиво»;
(2) подключить Figma через MCP — тогда Claude может опираться на реальные компоненты и переменные
дизайн-системы, а не изобретать свои.

### 🎯 Зачем тебе

Если у тебя уже есть закреплённая палитра/шрифты (как в этом самом боте — фон, акцент-терракота,
моно-шрифт для навигации), явное перечисление констант в промпте экономит 2-3 итерации правок
«не тот оттенок». Figma MCP полезен, когда дизайн уже живёт в Figma и код должен ему соответствовать
1:1, а не «примерно похоже».

### 💻 Минимальный пример

Промпт без токенов: «сделай карточку товара» → шаблонный Material-подобный вид.
Промпт с токенами: «карточка товара, фон #111110, текст #F5F3EC, акцент #d97757, моно-шрифт для
цены, обычный sans для описания» → предсказуемый результат с первого раза.

### ⚠️ Грабли

Каждый пользователь Figma MCP аутентифицируется отдельно, даже если артефакт или конфигурация
общие — доступ к дизайн-файлу не «наследуется» автоматически другим человеком. Начинать стоит с
малого: одна карточка/один компонент за раз, а не «сгенерируй весь дизайн-язык», иначе Claude
начинает угадывать недостающие токены вместо того чтобы честно опираться на реальные переменные.
