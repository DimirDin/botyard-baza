### ❓ Что это

Artifacts как генератор интерфейсов — Claude превращает текстовое описание идеи в рабочий интерфейс:
не мокап-картинку, а реально функционирующий React-компонент с состоянием, обработчиками событий,
условным рендерингом. Песочница артефакта поддерживает большинство стандартных возможностей React и
популярных библиотек (графики, иконки) без необходимости настройки окружения — но это именно
песочница с определёнными физическими границами, не полноценная среда разработки со свободным
доступом к любому npm-пакету и сети.

### 🎯 Зачем тебе

Быстрое прототипирование идеи интерфейса, когда важнее увидеть и покликать концепцию за минуту, чем
сразу писать продакшен-код — проверка гипотезы «как это вообще должно выглядеть и работать», прежде
чем вкладываться в полноценную реализацию в реальном проекте. Особенно полезно для не-разработчиков,
которым нужно быстро визуализировать идею для обсуждения с командой.

### 💻 Минимальный пример

```
Сделай интерфейс для трекера привычек: список привычек с чекбоксами на сегодня,
счётчик streak дней подряд, кнопка добавления новой привычки
```

Результат — реально кликабельный компонент с локальным состоянием (чекбоксы переключаются, счётчик
меняется), а не статичная картинка макета.

### ⚠️ Грабли

- **Песочница не имеет доступа к реальному бэкенду/сети** по умолчанию — состояние живёт только в
  рамках текущей сессии артефакта, при обновлении страницы обычно сбрасывается, это прототип, не
  подключённое к реальным данным приложение.
- **Не любая npm-библиотека доступна** — песочница поддерживает ограниченный, хотя и достаточно
  широкий, набор зависимостей, экзотические или очень специфичные пакеты могут не работать.
- **Сгенерированный «рабочий» интерфейс не равен продакшен-качеству кода** — доступность,
  производительность на реальных объёмах данных, edge cases обработки ошибок обычно требуют
  дополнительной проработки при переносе в реальный проект (см. Export and publish).
