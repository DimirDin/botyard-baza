### ❓ Что это

Code execution tool даёт Claude песочницу для вычислений: анализ данных, графики, парсинг файлов,
статистика, генерация файлов на выходе. Песочница изолирована, без доступа к интернету по умолчанию,
только файлы, явно переданные в сессию.

Отличие от Bash tool: Bash обычно даёт доступ к реальной инфраструктуре (репозиторий, CI), а
code execution — про самодостаточные вычисления внутри одного ответа, без побочных эффектов вне
песочницы.

### 🎯 Зачем тебе

«Посчитай медиану по CSV», «построй график распределения», «проверь гипотезу на данных» — задачи,
где текстовое рассуждение модели «в уме» ненадёжно, а реальный запуск кода даёт детерминированный
правильный результат.

### 💻 Минимальный пример

```json
{
  "tools": [{ "type": "code_execution_20250522", "name": "code_execution" }],
  "messages": [{ "role": "user", "content": "Посчитай P95 задержки из приложенного CSV" }]
}
```

Модель сама генерирует и исполняет `pandas.read_csv(...).quantile(0.95)`, видит реальный вывод и
уже на его основе формулирует ответ.

### ⚠️ Грабли

- **Не путать с programmatic tool calling** — code execution про вычисления модели, programmatic
  tool calling — про вызов твоих тулов из кода в той же песочнице.
- **Нет сохранения состояния между сообщениями по умолчанию** — уточняй режим сохранения файлов.
- **Лимиты по времени и памяти** — не для тяжёлых пайплайнов, только лёгкий-средний анализ.
- **Изоляция от интернета — намеренное ограничение, не временный недочёт** — если задача требует
  сходить за внешними данными (курс валюты, свежая цена), code execution сам по себе не решает это,
  нужна отдельная передача этих данных в сессию или комбинация с web search tool.
