### ❓ Что это

Computer use — Claude управляет десктопом как человек: получает скриншот, решает, куда кликнуть или
что напечатать, отправляет действие, получает новый скриншот. Модель не имеет доступа к DOM или
accessibility-дереву — она «видит» пиксели и целится в них, как человек мышкой.

Более общий, но и более медленный/дорогой инструмент, чем специализированные браузерные API — зато
работает с любым приложением с визуальным интерфейсом, включая десктопный софт без API.

### 🎯 Зачем тебе

Автоматизация legacy-десктопных приложений без API, end-to-end тест UI, заполнение форм в системах,
которые нельзя интегрировать иначе. Для веб-задач обычно эффективнее выделенный браузерный инструмент,
Computer use — универсальный fallback там, где такого инструмента нет.

Стоит явно понимать место Computer use в спектре инструментов автоматизации: DOM-aware браузерное
расширение (см. Claude в Chrome) быстрее и точнее для веба именно потому, что видит структуру
страницы, а не только пиксели. Computer use — на ступень более общий и более медленный инструмент,
который работает вообще с чем угодно на экране, включая приложения без какого-либо программного
интерфейса — цена этой универсальности в задержке на каждый шаг и меньшей точности попадания в
маленькие элементы интерфейса.

### 💻 Минимальный пример

```json
{
  "tools": [{ "type": "computer_20250124", "name": "computer", "display_width_px": 1280, "display_height_px": 800 }],
  "messages": [{ "role": "user", "content": "Открой настройки и включи тёмную тему" }]
}
```

Модель шлёт действия вида `{"action": "screenshot"}`, `{"action": "left_click", "coordinate": [x, y]}`,
`{"action": "type", "text": "..."}` — раннер их исполняет и присылает новый скриншот.

### ⚠️ Грабли

- **Beta-статус и заметная латентность** — каждый шаг требует round-trip с тяжёлым скриншотом.
- **Точность клика зависит от разрешения/масштабирования экрана** — рассинхрон DPI приводит к
  промахам мимо маленьких кнопок.
- **Требует такой же осторожной изоляции, как Bash tool** — не давай агенту доступ к десктопу с
  реальным банкингом/трейдингом без явного human-in-the-loop.
