### ❓ Что это
Набор из пяти независимых рычагов, каждый из которых снижает эффективную стоимость запросов к API без потери качества ответа: prompt caching, Batch API, роутинг по моделям, обрезка контекста и выбор глобального эндпоинта.

### 🎯 Зачем тебе
Разница между «сколько стоит токен по прайсу» и «сколько реально платит команда» может отличаться в разы — именно эти рычаги определяют итоговый счёт, а не номинальная цена модели.

### 💻 Минимальный пример
1. Кэшируй статичные системные промпты и документацию (`cache_control`)
2. Уводи фоновые несрочные задачи в Batch API (−50%)
3. Роутинг: Haiku для классификации → Sonnet для генерации → Opus/Fable только для сложного шага
4. Не таскай в контексте то, что не нужно для текущего запроса
5. Используй глобальный эндпоинт вместо регионального, если не требуется резидентность данных

### ⚠️ Грабли
Все пять рычагов дают эффект только при системном применении. Включить кэширование, но продолжать гонять всё на самой дорогой модели — не даст ожидаемой экономии.
