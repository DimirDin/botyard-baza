### ❓ Что это

Fast mode — режим, оптимизированный под минимизацию задержки первого токена и общей скорости
генерации, ценой части возможной глубины рассуждения. Тот же переключатель, что в Claude Code
называется Fast mode для Opus — быстрее той же модели, а не деградация до меньшей.

### 🎯 Зачем тебе

Интерактивные сценарии, где отзывчивость важнее последнего процента качества — autocomplete-подобные
фичи, live-чат с частыми короткими репликами. Не имеет смысла для тяжёлых аналитических задач, где
важна глубина, а не скорость первого токена.

### 💻 Минимальный пример

В Claude Code переключается через `/fast`. В API — соответствующий параметр запроса; ключевая
практика — не включать глобально по умолчанию, а тестировать A/B на латентность/качество.

### ⚠️ Грабли

- **«Быстрее» не значит «дешевле»** — отдельная ось от pricing, не путай экономию времени с
  экономией денег.
- **Не всегда доступен для всех моделей** — сверяйся с актуальным списком.
- **Замер latency должен быть end-to-end** — выигрыш теряется, если задержка в основном на твоём
  бэкенде.
