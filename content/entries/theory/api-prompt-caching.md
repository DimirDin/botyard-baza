### ❓ Что это

Prompt caching позволяет размечать блоки контекста (`cache_control: {"type": "ephemeral"}`), которые
Anthropic кэширует на своей стороне на ограниченное время. Повторное чтение закэшированного префикса
стоит до 90% дешевле обычного input-токена — модель не пересчитывает attention над тем же большим
системным промптом/кодовой базой при каждом запросе подряд.

### 🎯 Зачем тебе

Большой стабильный контекст, повторяющийся между запросами: system prompt с полной инструкцией
агента, вся кодовая база у coding-агента, длинная история в саппорт-боте. Без кэша каждый следующий
вопрос платит за полное чтение этого блока заново.

### 💻 Минимальный пример

```json
{
  "system": [
    { "type": "text", "text": "Ты агент проекта botyard-baza...", "cache_control": { "type": "ephemeral" } }
  ],
  "messages": [...]
}
```

Кэшируемый блок должен быть **префиксом** — если что-то меняется до размеченного блока, кэш не
сработает.

### ⚠️ Грабли

- **Динамические данные в начале system-промпта убивают кэш для всего, что после них** — переноси
  их в конец, после закэшированного блока.
- **Кэш не бесплатен на запись** — первый запрос стоит чуть дороже, экономия со второго попадания.
- **TTL ограничен** — большая пауза между запросами снова платит полную цену.
