### ❓ Что это

Service tiers определяют приоритет обработки запроса под нагрузкой: `priority` — гарантированный
доступ к капасити с минимальной задержкой (доплата), `standard` — обычная обработка по умолчанию,
`batch` — асинхронная обработка с окном до суток по вдвое меньшей цене. Это ортогональная ось к
выбору модели — тир определяет приоритет обслуживания, а не качество ответа.

### 🎯 Зачем тебе

Продакшен с пользователями, ждущими ответ прямо сейчас — кандидат на `priority` в часы пиковой
нагрузки. Фоновые задачи (генерация отчётов ночью, разметка данных) — прямая дорога в `batch`.

### 💻 Минимальный пример

```json
{ "model": "claude-sonnet-5", "service_tier": "priority", "messages": [...] }
```

Практика: `standard` как дефолт, `priority` — точечно для latency-критичных эндпоинтов, `batch` —
явно для всего, что можно обработать пачкой ночью.

### ⚠️ Грабли

- **`priority` не ускоряет саму генерацию** — влияет на очередь до начала генерации при перегрузке.
- **Нет глобального тира по умолчанию для всего аккаунта** — это параметр каждого запроса.
- **Batch — отдельный эндпоинт**, не флаг на обычном `/v1/messages` (создание задачи, поллинг,
  забор результатов).
