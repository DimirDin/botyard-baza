### ❓ Что это

Text editor tool (`str_replace_based_edit_tool`) — точечное редактирование файлов: вместо
перегенерации всего файла ради одной правки, модель вызывает `view` (посмотреть содержимое),
`str_replace` (заменить фрагмент), `create` (создать файл), `insert` (вставить по номеру строки).

Это механизм, на котором в Claude Code работает Edit tool — точное совпадение старой строки
обязательно, иначе операция отклоняется.

### 🎯 Зачем тебе

Правка одной функции в файле на 2000 строк не должна стоить регенерации всех 2000 строк. Экономия
токенов кратная, а главное — надёжность: `str_replace` либо находит точное совпадение, либо явно
падает с ошибкой — в отличие от полной регенерации, где тихая потеря куска кода проскальзывает
незамеченной.

### 💻 Минимальный пример

```json
{
  "tools": [{ "type": "text_editor_20250728", "name": "str_replace_based_edit_tool" }],
  "messages": [{ "role": "user", "content": "Исправь опечатку в handleSubmit в App.tsx" }]
}
```

Модель вызывает `view`, находит блок, затем `str_replace` со старым и новым текстом. Если старая
строка встречается больше раза — тул должен вернуть ошибку неоднозначности.

### ⚠️ Грабли

- **Совпадение должно быть побайтовым**, включая отступы — частая причина падений: модель «помнит»
  код чуть иначе, чем он выглядит после правок другим инструментом.
- **Не даёт diff/патч сам по себе** — панель ревью с подсветкой строится поверх результата.
- **Требует предварительного `view`** — без просмотра замена скорее всего не совпадёт.
