### ❓ Что это

Ключевая инженерная статья Anthropic, формализующая терминологию, ставшую стандартом индустрии:
**workflow** — заранее прописанный код, оркеструющий LLM и тулы по фиксированным путям, и **agent** —
система, где сама модель динамически решает, какие шаги предпринять, опираясь на обратную связь от
среды. Главный тезис: тяжёлые агентные фреймворки часто добавляют слои абстракции, усложняющие дебаг,
без пропорционального выигрыша — простые композируемые паттерны поверх API чаще надёжнее.

#### Пять паттернов из статьи

1. **Prompt chaining** — последовательность LLM-вызовов с программными проверками между шагами.
2. **Routing** — классификация входа и направление в один из специализированных путей.
3. **Parallelization** — секционирование (независимые подзадачи) или голосование (несколько попыток).
4. **Orchestrator-workers** — центральная модель динамически делегирует суб-агентам.
5. **Evaluator-optimizer** — один вызов генерирует, другой критикует, цикл до критерия остановки.

### 🎯 Зачем тебе

Правильная точка старта перед тем, как тянуть в проект тяжёлый агентный фреймворк. Если задача
укладывается в fixed workflow — простой prompt chaining на голом API надёжнее и предсказуемее по
латентности и стоимости, чем «настоящий» автономный агент.

### 💻 Минимальный пример

```python
category = classify(user_message)  # LLM-вызов, дешёвая модель
if category == "billing":
    return handle_billing(user_message)
elif category == "technical":
    return handle_technical(user_message)
```

Это уже «агентный паттерн» в терминологии статьи — явный, читаемый код, не абстракция фреймворка.

### ⚠️ Грабли

- **«Агент» не значит «лучше»** — для линейной задачи автономный агент добавляет непредсказуемость
  и стоимость без выгоды.
- **Фреймворки прячут промпты за абстракциями**, усложняя отладку конкретного плохого ответа.
- **Evaluator-optimizer без критерия остановки зацикливается** — закладывай максимум итераций.
