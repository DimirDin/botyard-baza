### ❓ Что это

Механизм API, автоматически вычищающий устаревшие результаты вызовов инструментов (`tool_result`) из
окна по мере продвижения диалога, вместо того чтобы держать их все накопленным грузом до конца сессии.
Если агент 20 шагов назад прочитал файл через тул, а сейчас файл уже 5 раз перечитан заново — старые
версии результата почти наверняка больше не нужны в контексте.

### 🎯 Зачем тебе

Особенно заметный эффект на агентах вроде Claude Code, делающих десятки вызовов Read/Bash/Grep за
сессию — без context editing история этих вызовов быстро становится доминирующей частью контекста,
вытесняя более свежую информацию.

### 💻 Минимальный пример

Устаревшие `tool_result` блоки заменяются короткими плейсхолдерами вида «[результат Read файла X
удалён из контекста, файл был прочитан повторно позже]», сохраняя целостность истории вызовов без
веса полного содержимого.

### ⚠️ Грабли

- **Не для tool-результатов, остающихся единственным источником факта** — агрессивная очистка рискует
  потерять единственную копию нужной информации.
- **Комбинируется, но не заменяет compaction** — дополняющие механизмы одной проблемы переполнения.
- **Прозрачность для дебага снижается** — восстановить полную картину сложнее при вычищенной истории.
