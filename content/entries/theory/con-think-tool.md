### ❓ Что это

Приём: добавить агенту простой инструмент `think`, вызов которого даёт модели место записать
структурированное рассуждение посреди задачи — без побочных эффектов, просто явный шаг «пауза на
подумать» в истории диалога.

#### Отличие от extended thinking

Think tool появился раньше extended thinking и решал ту же проблему проще, но менее интегрированно:
явный тул, который агент сам решает вызвать. Extended thinking встроен в сам механизм генерации и не
требует отдельного тула.

### 🎯 Зачем тебе

Исторически улучшал следование политикам и консистентность решений при минимальной реализации. Сегодня
при уже используемом extended/adaptive thinking отдельный think tool в большинстве случаев избыточен.
Остаётся полезен там, где extended thinking недоступен или важно явно видеть паузу как отдельное
событие в логе.

Показательный пример того, как быстро развивается эта область: приём, ещё недавно бывший
рекомендуемой лучшей практикой, стал по большей части избыточным с появлением встроенного механизма,
решающего ту же задачу нативнее. Статья остаётся полезной не как актуальная рекомендация «добавь этот
тул», а как иллюстрация самой проблемы — агенту иногда нужно явное место остановиться и подумать
перед действием с побочным эффектом, вне зависимости от того, каким конкретно механизмом это сейчас
реализовано.

### 💻 Минимальный пример

```json
{
  "name": "think",
  "description": "Используй, чтобы обдумать сложное решение перед действием. Не меняет состояние.",
  "input_schema": { "type": "object", "properties": { "thought": { "type": "string" } }, "required": ["thought"] }
}
```

В системном промпте: «Перед вызовом тулов с побочными эффектами вызови `think`, чтобы проверить план».

### ⚠️ Грабли

- **Устаревающий паттерн при наличии extended thinking** — сначала проверь, не закрывает ли задачу
  нативный thinking-режим.
- **Тул без side-эффектов легко забыть проверить в тестах** — разработчики не замечают, что агент
  перестал его вызывать.
- **Не подменяет реальную валидацию** — для критичных проверок нужна настоящая программная проверка.
