### ❓ Что это
**Skills** — папки с инструкциями, скриптами и шаблонами, которые Claude подключает динамически под конкретную задачу. Skill не меняет саму модель — он даёт ей процедурное знание: как именно выполнить конкретный тип работы повторяемым способом (например, создать презентацию по фирменному стилю компании или заполнить Excel-отчёт по шаблону).

У Anthropic есть готовые Skills для типовых офисных задач (PowerPoint, Excel, Word, PDF) — они уже включены в claude.ai, не нужно ничего доустанавливать. Можно и создавать свои собственные, если у тебя есть повторяющаяся задача со своим форматом.

### 🎯 Зачем тебе
Без Skill каждый раз приходится заново объяснять Claude в промпте, как именно должен выглядеть результат — какой формат, какие разделы, какой стиль оформления. Skill один раз фиксирует эти правила, и дальше просто включается по названию задачи — не нужно пересказывать инструкцию с нуля в каждом диалоге.

Это особенно полезно для команд: если у компании есть свой формат отчёта или брендбук, Skill с этими правилами можно один раз настроить и переиспользовать — результат будет единообразным независимо от того, кто из команды просит Claude выполнить задачу.

### 💻 Как это выглядит на практике
Чтобы посмотреть доступные Skills — в настройках аккаунта раздел «Customize» → «Skills», кнопка «+» → «Browse skills» открывает каталог. На скриншоте выше — меню управления уже подключённым Skill: можно попробовать его в чате («Try in chat»), скачать, отредактировать через Claude или удалить, если больше не нужен.

Использование в диалоге не требует специального синтаксиса — Claude сам определяет, что задача подходит под конкретный Skill, и подключает нужные инструкции автоматически. Например, попросив «сделай презентацию по этим тезисам в фирменном стиле», при подключённом Skill для PowerPoint получаешь готовый файл нужного формата, а не просто текстовый план слайдов.

Разница между Skills и MCP-коннекторами (о них — следующий урок): MCP даёт Claude доступ к внешним инструментам и сервисам, а Skills учат Claude, как именно этими инструментами эффективно пользоваться. Это дополняющие, а не конкурирующие механизмы.

### ⚠️ Частая ошибка новичка
Пытаться вручную «включить» Skill командой или упоминанием в промпте, как если бы это был режим или флаг — Skills подключаются автоматически по смыслу задачи, а не по явному запросу пользователя. Если Skill не подключился — стоит переформулировать задачу ближе к тому, для чего Skill предназначен, а не искать способ вызвать его напрямую.
