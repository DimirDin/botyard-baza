### ❓ Что это
Установка Claude Code — одна команда в терминале, без отдельного лицензионного ключа сверх обычной подписки Claude:
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
(на Windows — аналогичный `irm https://claude.ai/install.ps1 | iex` через PowerShell). После установки команда `claude`, запущенная внутри папки проекта, открывает интерактивную сессию — Claude Code сразу видит структуру файлов в текущей директории, ничего дополнительно указывать не нужно.

### 🎯 Зачем тебе
Первый запрос задаёт тон всей работе с инструментом: если сформулировать задачу конкретно, с первого раза получаешь рабочий результат и доверие к инструменту растёт; если сформулировать расплывчато — первое впечатление будет «он не понимает, что мне нужно», хотя дело в формулировке, а не в возможностях агента.

### 💻 Как это выглядит на практике
Заходишь в папку проекта и запускаешь сессию:
```bash
cd /путь/к/проекту
claude
```
Дальше просто пишешь задачу текстом, как в обычном чате — но результат совсем другой: Claude Code читает реальные файлы проекта и предлагает конкретные правки, а не абстрактный совет.

Хороший первый запрос для знакомства с инструментом — что-то маленькое и проверяемое: «покажи структуру этого проекта и объясни, за что отвечает каждая папка» или «найди все TODO-комментарии в коде и покажи список». Такие задачи безопасны (ничего не меняют) и сразу показывают, как Claude Code исследует кодовую базу.

Первая правка обычно выглядит так:
1. Пишешь: «добавь docstring этой функции» (указав файл или вставив код).
2. Claude Code находит функцию, предлагает диф — что именно добавится.
3. Проверяешь и подтверждаешь (на первый запуск подтверждение запрашивается почти на каждое действие — это нормально, подробнее на уроке про Permissions).

Полезно с самого начала знать про **слэш-команды** — служебные команды с префиксом `/` (например `/clear` для очистки контекста сессии, `/help` для списка возможностей) — это не то же самое, что обычный текстовый запрос к модели.

### ⚠️ Частая ошибка новичка
Начинать знакомство сразу с крупной задачи на боевом проекте («перепиши всю авторизацию») вместо маленькой безопасной («покажи, как устроена авторизация сейчас»). Первые запросы стоит выбирать так, чтобы легко было проверить результат и понять логику работы инструмента, прежде чем доверять ему более рискованные изменения.
