### ❓ Что это
**Claude Code** — официальный агентный CLI-инструмент от Anthropic. В отличие от claude.ai, он работает прямо в терминале, видит файлы твоего проекта, может сам их читать, менять, запускать команды и коммитить в git. Это уже не просто «чат, который пишет код» — это агент, который выполняет реальные действия в твоей файловой системе, шаг за шагом, пока не решит задачу.

Ключевое отличие от обычного чата: Claude Code не просто советует, что сделать, а делает это сам — читает файл, находит нужное место, правит, запускает тесты, проверяет результат, и если что-то не сработало — пробует иначе, без твоего постоянного участия в каждом шаге.

### 🎯 Зачем тебе
Если ты пишешь код, Claude Code избавляет от постоянного копирования кусков туда-сюда между чатом в браузере и редактором. Ты просто говоришь ему, что сделать — он сам находит нужные файлы, правит их и показывает диф (наглядное сравнение «было / стало») для проверки перед тем, как изменения окончательно применятся.

Это меняет сам процесс работы: вместо «спроси — скопируй ответ — вставь в редактор — проверь — вернись спросить, что не так» получаешь «сформулируй задачу — Claude Code сам сделал — ты проверил диф — одобрил или попросил поправить». Особенно заметная экономия времени на рутинных, но объёмных задачах — массовый рефакторинг, добавление тестов, обновление зависимостей по всему проекту.

### 💻 Как это выглядит на практике
Установка одной командой:
```bash
curl -fsSL https://claude.ai/install.sh | bash
```
Дальше в терминале, находясь в папке проекта:
```
cd /путь/к/проекту
claude
```
И общаешься с ним так же текстом, но результат — реальные изменения файлов в этой папке, а не просто ответ в чате.

![](/guide/3/guide-claude-code-intro-detail.svg)

Типичный рабочий цикл выглядит так:
1. Формулируешь задачу текстом: «добавь валидацию email в форму регистрации».
2. Claude Code сам находит релевантные файлы — сканирует структуру проекта, а не спрашивает тебя, где что лежит.
3. Вносит правки и показывает диф — что именно изменилось, файл за файлом.
4. Ты проверяешь и одобряешь (или просишь переделать конкретную часть) — при первом использовании подтверждения запрашиваются на каждое действие, подробнее об этом на следующем уроке про Permissions.
5. При необходимости Claude Code сам запускает тесты или линтер, чтобы убедиться, что ничего не сломалось.

Claude Code также умеет работать с git напрямую — создавать коммиты с осмысленными сообщениями, смотреть историю изменений, разбираться в конфликтах при слиянии веток.

### ⚠️ Частая ошибка новичка
Давать агенту слишком общую задачу без структуры проекта («сделай сайт лучше») — Claude Code работает намного точнее, если у него есть конкретика: какой файл, что именно поменять, какой результат ожидается. Расплывчатая задача заставляет агента гадать о твоих намерениях, и результат может оказаться совсем не тем, что ты имел в виду.

Вторая частая ошибка — не проверять диф перед принятием изменений, особенно на больших правках. Даже хороший агент иногда делает больше, чем просили, или интерпретирует задачу неожиданным образом — быстрый просмотр изменений перед подтверждением экономит время на исправлении ошибок позже. Об этом подробнее — следующий урок, про CLAUDE.md и про то, как дать агенту нужный контекст заранее.
