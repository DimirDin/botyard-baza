### ❓ Что это
**CLAUDE.md** — обычный markdown-файл в корне репозитория. Claude Code автоматически читает его в начале каждой сессии, чтобы узнать структуру проекта, команды сборки/тестов и правила стиля кода. Это не какой-то специальный формат с секретным синтаксисом — обычный markdown, который ты бы написал для человека-новичка в проекте, только гораздо короче и суше.

Файл можно (и стоит) держать в git вместе с остальным кодом — тогда его подхватывают все, кто работает над проектом с Claude Code, а не только ты лично.

### 🎯 Зачем тебе
Без этого файла агент каждый раз заново гадает, каким пакетным менеджером пользуется проект, как запускать тесты, какой стиль кода принят — и либо угадывает верно, либо тратит впустую шаг на пробу и ошибку. Один раз описав это, ты экономишь время и токены на каждой сессии — и получаешь код в едином стиле, а не такой, какой агент придумал по своему усмотрению в этот конкретный раз.

Чем больше в команде людей работает с Claude Code над одним репозиторием, тем ценнее этот файл — он становится единым источником правды про конвенции проекта, вместо того чтобы каждый настраивал агента под себя устно.

### 💻 Как это выглядит на практике
```markdown
## Build & Test
- Install: `npm install`
- Dev server: `npm run dev`
- Tests: `npm test`

## Code Style
- TypeScript, строгая типизация
- Компоненты — функциональные, с хуками
```
Достаточно положить такой файл в корень репозитория — Claude Code подхватит его сам, без дополнительных настроек, при следующем же запуске `claude` в этой папке.

![](/guide/3/guide-claude-md-detail.svg)

Что имеет смысл включать в CLAUDE.md, помимо базовых команд:
- **Архитектурные решения, которые не менять без явного запроса** — например, «не трогай логику гейта в auth.py без сверки с документацией».
- **Специфичные для проекта факты** — какая СУБД используется, какие переменные окружения обязательны, где искать конфигурацию.
- **Ссылки на внешнюю документацию**, если полная спецификация проекта живёт не в репозитории, а отдельно.

Файл можно (и стоит) обновлять по мере роста проекта — как только Claude Code делает что-то не так из-за незнания конвенции, это сигнал добавить соответствующую строчку в CLAUDE.md, чтобы больше не повторять объяснение вручную.

### ⚠️ Частая ошибка новичка
Раздувать CLAUDE.md на сотни строк с историей проекта и общими рассуждениями. Большой файл забивает контекстное окно и увеличивает стоимость каждого запроса — пиши только сухие факты и команды, а не документацию для людей, которую и так можно найти в README.

Вторая ошибка — писать файл один раз и никогда не обновлять, даже когда проект меняется. Устаревший CLAUDE.md, который советует давно отменённые команды или ссылается на удалённые файлы, хуже, чем его отсутствие — агент будет честно следовать написанному, даже если это уже неверно.
