### ❓ Что это
**CLAUDE.md** — обычный markdown-файл в корне репозитория. Claude Code автоматически читает его в начале каждой сессии, чтобы узнать структуру проекта, команды сборки/тестов и правила стиля кода.

### 🎯 Зачем тебе
Без этого файла агент каждый раз заново гадает, каким пакетным менеджером пользуется проект, как запускать тесты, какой стиль кода принят. Один раз описав это, ты экономишь время и токены на каждой сессии — и получаешь код в едином стиле.

### 💻 Как это выглядит на практике
```markdown
## Build & Test
- Install: `npm install`
- Dev server: `npm run dev`
- Tests: `npm test`

## Code Style
- TypeScript, строгая типизация
- Компоненты — функциональные, с хуками
```
Достаточно положить такой файл в корень репозитория — Claude Code подхватит его сам, без дополнительных настроек.

### ⚠️ Частая ошибка новичка
Раздувать CLAUDE.md на сотни строк с историей проекта и общими рассуждениями. Большой файл забивает контекстное окно и увеличивает стоимость каждого запроса — пиши только сухие факты и команды, а не документацию для людей.
