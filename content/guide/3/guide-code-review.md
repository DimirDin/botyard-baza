### ❓ Что это
Claude Code можно попросить проревьюить код так же, как попросить его написать — разница только в задаче: не «сделай», а «проверь и укажи на проблемы». Ревью может касаться локальных незакоммиченных изменений (`git diff`), конкретного файла, всего пул-реквеста через интеграцию с GitHub, или даже чужого кода, который ты просто открыл в терминале.

Это дополняет, а не заменяет ревью человека — Claude Code хорошо ловит очевидные технические проблемы (пропущенную обработку ошибок, потенциальные race conditions, несоответствие конвенциям проекта), но не заменяет понимание бизнес-контекста, которое есть у живого ревьюера в команде.

### 🎯 Зачем тебе
Первый проход ревью — рутинная, но необходимая часть разработки: заметить очевидные проблемы до того, как их увидит человек-ревьюер, экономит время всей команды и ускоряет цикл разработки. Особенно полезно перед тем, как выложить пул-реквест — прогнать код через Claude Code первым, поймать очевидное самому, и уже потом отдавать на ревью коллеге, который сможет сосредоточиться на более глубоких архитектурных вопросах.

### 💻 Как это выглядит на практике
Самый простой сценарий — попросить проверить незакоммиченные изменения прямо в текущей сессии: «прогляди git diff и укажи на проблемы» — Claude Code сам достаёт diff и анализирует его построчно.

Более формальный запрос — попросить сыграть роль строгого ревьюера, а не просто «посмотри код»: явно указать, что нужен придирчивый разбор, а не общая похвала. Разница в формулировке ощутимо влияет на глубину ревью — «посмотри, всё ли ок» почти всегда даёт более поверхностный ответ, чем «найди 5 конкретных проблем, даже незначительных».

Типичные вещи, которые Claude Code хорошо ловит при ревью:
- **Пропущенная обработка ошибок** — например, незакрытые try/except или проигнорированный код ответа API.
- **Несоответствие конвенциям проекта** — если в `CLAUDE.md` описан стиль кода, ревью сверяется именно с ним, а не с общими best practices.
- **Потенциальные проблемы безопасности** — SQL-инъекции, небезопасная десериализация, утечка секретов в лог.
- **Излишняя сложность** — код, который решает задачу таким образом, что его сложно поддерживать, даже если формально работает правильно.

### ⚠️ Частая ошибка новичка
Просить общее «посмотри, всё ли нормально» вместо конкретной задачи ревью — расплывчатый запрос почти всегда даёт расплывчатый, вежливый ответ без реальной пользы. Чем конкретнее сформулирован критерий («найди проблемы с обработкой ошибок», «проверь на соответствие CLAUDE.md»), тем полезнее результат.

Вторая ошибка — воспринимать одобрение от Claude Code как гарантию отсутствия проблем. Это первый фильтр, а не финальная инстанция — для критичного кода (платежи, авторизация, работа с личными данными) человеческое ревью всё равно обязательно.
