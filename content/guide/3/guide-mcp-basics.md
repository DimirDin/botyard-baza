### ❓ Что это
**MCP** (Model Context Protocol) — открытый стандарт связи между Claude и внешними инструментами. MCP-сервер «объявляет», какие у него есть инструменты (например, «прочитать таблицу из базы данных», «создать issue в GitHub», «отправить сообщение в Slack»), а Claude может их вызывать, не требуя отдельной кастомной интеграции под каждый сервис.

Технически MCP работает по клиент-серверной модели: MCP-клиент (Claude Code, Claude Desktop) обменивается сообщениями с MCP-сервером по стандартизированному протоколу. Ты как пользователь этого обмена не видишь — просто просишь что-то на естественном языке, а вызов нужного инструмента происходит внутри автоматически.

### 🎯 Зачем тебе
Без MCP каждую интеграцию пришлось бы писать с нуля — отдельный код для подключения к каждой базе данных, каждому API, каждому сервису. С готовым MCP-сервером, например, для Postgres, Claude сразу получает типизированные инструменты для чтения таблиц — установил и подключил, а не написал плагин самостоятельно.

Экосистема MCP-серверов растёт очень быстро — уже есть готовые серверы для большинства популярных баз данных, облачных хранилищ, систем управления проектами и коммуникационных платформ. Это означает, что в большинстве случаев тебе не придётся писать интеграцию самому — достаточно найти готовый сервер и подключить его.

### 💻 Как это выглядит на практике
Подключение MCP-сервера в Claude Code — одной командой:
```bash
claude mcp add имя-сервера -- npx -y <mcp-пакет>
claude mcp list   # проверить, что подключилось
```
После подключения в диалоге можно просто попросить: «покажи структуру таблицы users» — и Claude сам вызовет нужный инструмент MCP-сервера, получит результат и ответит на основе реальных данных, а не догадок.

![](/guide/3/guide-mcp-basics-detail.svg)

Несколько типичных категорий MCP-серверов, которые стоит знать:
- **Серверы для баз данных** — Postgres, MySQL, SQLite: чтение схем, выполнение запросов.
- **Серверы для файловых систем** — работа с файлами за пределами текущего проекта, если это разрешено настройками.
- **Серверы для внешних сервисов** — GitHub, Slack, Notion, Google Drive: чтение и создание контента напрямую из диалога.
- **Серверы для инфраструктуры** — Docker, Kubernetes: просмотр статуса контейнеров и логов (подробнее — на уровне 4).

Для командной работы список подключённых MCP-серверов можно зафиксировать в файле `.mcp.json` в корне репозитория — тогда все участники команды получают одинаковый набор инструментов автоматически, без ручной настройки каждым по отдельности.

### ⚠️ Частая ошибка новичка
Подключать непроверенные MCP-серверы из случайных источников. MCP-сервер — это реальный исполняемый код на твоей машине, который может иметь доступ к файлам, сети или учётным данным. Проверяй исходники перед подключением, особенно если сервер не от известного разработчика или официальной организации.

Вторая ошибка — подключать сразу много MCP-серверов «про запас», не используя большинство из них. Каждый подключённый сервер добавляет модели дополнительные инструменты для выбора, и слишком широкий набор редко используемых инструментов может немного снижать точность выбора нужного — разумнее подключать по мере реальной необходимости.
