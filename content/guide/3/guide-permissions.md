### ❓ Что это
Слой **Permissions** в Claude Code ограничивает, что агент может делать без твоего явного разрешения. По умолчанию он может читать и менять файлы текущего проекта, но для запуска bash-команд, установки пакетов или сетевых запросов спрашивает подтверждение.

### 🎯 Зачем тебе
Без этого контроля ошибка в сгенерированном скрипте может выполнить опасную команду или случайно перезаписать системный файл. Permissions создают безопасную «песочницу» вокруг .env, SSH-ключей и остальной системы за пределами проекта.

### 💻 Как это выглядит на практике
При попытке выполнить команду Claude Code спрашивает:
```
Запрашивает разрешение на выполнение:
$ npm install some-package

[Y] Разрешить один раз
[A] Разрешать всегда в этой сессии
[N] Отклонить
```
Ты решаешь в моменте — разово, на всю сессию, или отклонить вовсе.

### ⚠️ Частая ошибка новичка
Отключать ручные подтверждения «чтобы не отвлекаться», особенно в проекте с непроверенными сторонними зависимостями. Агент может случайно прочитать и выполнить вредоносный код из чужого пакета без твоего ведома — не отключай подтверждения там, где не уверен на 100% в безопасности зависимостей.
