### ❓ Что это
**Субагент** — отдельный AI-помощник внутри Claude Code, заточенный под конкретный тип задачи. У него свой собственный контекст (отдельное «окно памяти»), свой системный промпт и, если нужно, ограниченный набор доступных инструментов. Когда основная сессия встречает задачу, подходящую под описание субагента, она делегирует её — субагент работает независимо и возвращает только итоговый результат, а не весь процесс поиска.

Это принципиально отличается от того, чтобы просто попросить Claude Code «сделать ещё одну вещь» в той же сессии — субагент физически не засоряет основной контекст побочными деталями своей работы.

### 🎯 Зачем тебе
Если задача предполагает много «шумной» побочной работы — прочитать десяток логов, погуглить по документации, пройтись по файлам в поисках нужного паттерна — весь этот шум обычно не нужен в основном диалоге после того, как найден ответ. Субагент делает эту грязную работу в своём изолированном контексте и возвращает только выжимку — основной диалог остаётся чистым и сфокусированным на текущей задаче, не раздутым логами промежуточного поиска.

Это также способ контролировать расходы: рутинные субзадачи можно направить на более быструю и дешёвую модель (например, Haiku), оставляя основную сессию на более мощной модели только для действительно сложной части работы.

### 💻 Как это выглядит на практике
Субагент создаётся как обычный markdown-файл с системным промптом, описанием (по которому Claude Code решает, когда его вызывать) и опциональным ограничением набора доступных инструментов. Дальше он используется автоматически — стоит только сформулировать задачу так, чтобы она подходила под описание субагента, явно вызывать его по имени не обязательно.

Типичные кандидаты на отдельного субагента:
- **Ресёрчер по кодовой базе** — находит все места, где используется конкретная функция или паттерн, возвращает список без сохранения полного дерева поиска в основном контексте.
- **Ревьюер логов** — читает длинный вывод тестов или сборки, возвращает только суть: что упало и почему.
- **Специализированный по языку/фреймворку** — узкий системный промпт под конкретный стек (например, только Rust или только SQL-миграции), с доступом только к нужным для этой ниши инструментам.

Субагенты работают в рамках одной сессии. Если нужно запускать много независимых сессий параллельно и следить за ними из одного места — это уже другая механика (background agents), не субагенты как таковые.

### ⚠️ Частая ошибка новичка
Создавать субагента с слишком широким и расплывчатым описанием — Claude Code не поймёт, когда именно его вызывать, и либо будет делать это невпопад, либо вообще проигнорирует. Описание субагента должно быть таким же конкретным, как хороший промпт: «анализирует SQL-миграции на предмет несовместимых изменений», а не «помогает с базой данных».
