### ❓ Что это
Встроенные слэш-команды вроде `/clear` или `/help` — не единственные, которые понимает Claude Code. Можно создавать **свои** — markdown-файл в папке `.claude/commands/` становится доступен как `/имя-файла`, и при вызове его содержимое подставляется как промпт, с возможностью передать дополнительные аргументы.

Это простой, но мощный способ зафиксировать часто повторяющийся запрос один раз и вызывать его одним словом вместо того, чтобы каждый раз формулировать заново.

### 🎯 Зачем тебе
Если ты регулярно просишь Claude Code об одном и том же — например, «проверь этот файл на соответствие нашему стайлгайду и напиши тесты» — есть смысл зафиксировать это как команду `/check-and-test`, а не набирать длинную инструкцию каждый раз заново. Экономия не только в наборе текста: команда гарантирует, что формулировка задачи остаётся одинаковой каждый раз, а не немного плывёт от сессии к сессии.

Для команд это способ зафиксировать общий процесс — если у всей команды есть свой `/review-pr`, каждый участник получает одинаково тщательное ревью по одному и тому же чек-листу, а не по тому, что лично вспомнил попросить в моменте.

### 💻 Как это выглядит на практике
Файл `.claude/commands/review.md`:
```markdown
Проверь текущий diff на:
- Пропущенную обработку ошибок
- Несоответствие стилю проекта из CLAUDE.md
- Потенциальные проблемы безопасности

Дай список конкретных проблем, без общих слов.
```
После этого команда `/review` в терминале запускает именно этот промпт — без необходимости набирать инструкцию заново.

Команды поддерживают аргументы — можно передать что-то дополнительное при вызове (например, `/review src/auth.py`, чтобы уточнить конкретный файл), которое подставляется в заранее заданное место шаблона.

Команды, сохранённые в `.claude/commands/` в корне репозитория, попадают в git вместе с остальным кодом — весь, кто клонирует проект, получает те же самые команды автоматически, без ручной настройки.

### ⚠️ Частая ошибка новичка
Держать полезные команды только в личном (не проектном) конфиге — тогда ими пользуешься только ты, а не вся команда. Если команда полезна не только лично тебе, а для процесса в целом — стоит положить её в `.claude/commands/` внутри репозитория и закоммитить, а не оставлять в личных настройках.
