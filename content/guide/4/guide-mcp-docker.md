### ❓ Что это
Интеграция Claude Code с локальным Docker через MCP-сервер даёт агенту типизированные инструменты для сборки образов, просмотра статуса контейнеров, чтения логов и перезапуска — не выходя из текущей сессии.

### 🎯 Зачем тебе
Не нужно постоянно переключаться между терминалом с сессией Claude Code и соседней вкладкой с `docker compose logs -f`. Можно попросить агента самому найти причину падения контейнера по логам и поправить конфиг.

### 💻 Как это выглядит на практике
Подключение через официальную команду:
```bash
claude mcp add docker -- npx -y <docker-mcp-пакет>
claude mcp list
```
Для командной работы — зафиксируй в `.mcp.json` в корне проекта (коммитится в git, применяется у всей команды):
```json
{
  "mcpServers": {
    "docker": { "type": "stdio", "command": "npx", "args": ["-y", "<docker-mcp-пакет>"] }
  }
}
```

### ⚠️ Частая ошибка новичка
Оставлять агента в headless-режиме с доступом к Docker без присмотра. Проброс доступа к Docker-сокету фактически даёт root-права на хосте — деструктивная команда очистки может снести все локальные базы и кэши образов разом.
